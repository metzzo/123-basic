All extended widgets have all information that widget has.
A spacer is an invisible spacing block for alignment.

the widgets get drawn left->right until one is too
wide for the given dialog area
then the widget starts in a new line.

All widgets have a DDGui_handleXYZ function, which handles
the behaviour of the widget.
Here you get mx, my in coordinates relative to the widget's
top left corner.
b1 has the state of the left mouse button:
b1 = 0 - not pressed
b1 = 1 - just released the button (clicked)
b1 =-1 - button has just been pressed down


--------------------------------------------------------------
A BUTTON is a simple click-button. You check if it's been
clicked with IF DDgui_get("my_button_idCLICKED") THEN ...
when you set it's TEXT to SPR_B123, the sprite 123 is used
instead of a text
if you set the TEXT to SPR_C123, you have a color picker dialog
when you press it. To get the color write col=MID$((DDgui_get(id$+"TEXT"),5,32)
--------------------------------------------------------------
a SLIDER is, well a slider. It has a value from 0 to 1
( regardless of it's width ) stored in the TEXT.
You can drag it with the mouse
--------------------------------------------------------------
a TOOLBAR is a macro that creates a lot of buttons for you
you simply
DDgui_toolbar: id_buttons$[], id_sprites[], width_button, height
--------------------------------------------------------------
a CHECKBOX is a button, that toggles the checked state with
every click. Check the state with DDgui_get("my_check_idSELECT")
Attantion! That was CHECKED in older versions of DDgui!
--------------------------------------------------------------
a RADIO is a group of checkboxes, where only _one_ can be
checked at a time. If you add/remove checkboxes, the height
changes automatically.
--------------------------------------------------------------
FILE is a button, that has a disk-image. If you click it, you get
a filerequester. The filename is drawn as text
Get the filename with DDgui_get$("file_btn_idTEXT")
If no filename / invalid filename, the text is empty
--------------------------------------------------------------
TEXT and SINGLETEXT is a text box, where the user can type things,
select text, overwrite, insert and so forth.
a text needs the focus to be able to get keyboard messages
focus is gained by clicking once with the mouse
you can get/set the focus with DDgui_set("FOCUS", "my_id")

On PocketPC and GP2X, when you click it, DDgui performs a
DDgui_input$ for you. This way you get an iPhone like
keyboard that can be used with the stylus.
You can DEBUG this behaviour by setting
DDGUI_AUTO_INPUT_DLG = TRUE

The SINGLETEXT is a text box that allows only single line input.
and uses the tab key to forward the focus to the next widget.
--------------------------------------------------------------
a LIST is the same as a radio, but you can scroll through
many list items, the height you gave does not change.
--------------------------------------------------------------
a TAB is a button row, where you can activate one of them.
Afterwards DDgui will DDgui_hide/DDgui_unhide the widgets
within the tab.
Use DDgui_selecttab
you create a tab with a captions$ like:
name1,wd_1,wd2|name2,wd_3,wd_4
where name1,name2 are the texts for each of the the 2 tabs
wd_1,wd_2 are the widgets to show for tab1
wd_3,wd_4 are the widgets to show for tab2
--------------------------------------------------------------
a USER widget is just a rectangular placeholder
that calls a user defined function of prototype
DDgui_userfunction.
PROTOTYPE DDgui_userfunction: id$, verb$, BYREF info$
The GUI will call this function with the verb$ set to what the
widget is notified of.
You can send user defined messages to such a widget with the
command: DDgui_signal(). The info$ parameter is byref, meaning
you have a way to return things with that method.

Possible notifications, that DDgui sends are:
verb$, info$, meaning
INIT "" The widget was created
allocate internal TYPEs or images now.
DESTROY "" The widget was destroyed.
(DDgui_popdialog) - free resources.
CLICKED "XXXX,YYYY,BB" Left button was up. X,Y,B are mouse
coords in widget space.
DRAW "" Readraw the widget now.
x=DDgui_get(id$, "X")
w=DDgui_get(id$, "WIDTH")
h=DDgui_get(id$, "HEIGHT")
Viewport is set to the widget's rectangle already.

--------------------------------------------------------------
HELPER FUNCTIONS:
--------------------------------------------------------------
DDgui_msg: text$, yes_no
display a messagebox with the text
if yes_no is set it returns 1 i "yes", 0 if "no"
was clicked

DDgui_insertitem(id$, text$, insert_before)
let's you insert a item to a radio/list
if you want to append, use insert_before=-1

DDgui_deleteitem(id$, index)
removes an item from a radio/list
index= -1 removes them all

DDgui_getitemtext$(id$, index)
get the word of a radio/list's item
index=-1 get's the currently selected item

DDGui_colordgl(color%)
a simple dialog that let's you edit a color.
it returns the changed color.

DDGui_input$
a INPUT dialog with an onscreen keyboard.
Very useful for PocketPC devices w/o keyboard.
DDgui uses this on POCKETPC and GP2X platforms
if you tip on a non-READONLY text box.
--------------------------------------------------------------
Values you can DDgui_get() or DDgui_set():
--------------------------------------------------------------

### dialog/global: - widget$="" ###
MOVING - Indicates the dialog is currently moved.
XPOS, YPOS, WIDTH, HEIGHT - dialog dimensions
SCALEABLE - can scale dialog
MOVEABLE - can move dialog
MOVING - dialog is currently being moved
SCALEING - dialog is currently being resized
COL_BRIGHT - bright color
COL_NORM - normal color
COL_HOVER_BRIGHT - bright color when under mouse
COL_HOVER_NORM - normal color when under mouse
INKEY - a character, that is used instead of INKEY$
useful for button as a keyboard mapping.

--------------------------------------------------------------
### widget: ### - this is a set of states that any widget has
XPOS, YPOS - position, globally on screen
WIDTH
HEIGHT
HOVER 1/0
TEXT "text"
CLICKED 1/0
READONLY - if set, the user can't change the value
SCROLL - intern use
SCROLLMAX - intern use
HIDE - do not show/handle (use DDgui_hide/DDgui_unhide)
ALIGN - -1=left, 0=center, 1=right. Center only in the remaining space of that line!
TIPTEXT - optional tooltip text, that apears, when mouse hovers
over this item
--------------------------------------------------------------
### spacer: ###
--------------------------------------------------------------
### button: ###
CLICKED 1/0
SELECT 1/0 - indicate that this button is "pressed/highlit"
READONLY1/0 - button is disabled
--------------------------------------------------------------
### slider: ###
TEXT 0...1
CLICKED inform about mouse is moving slider
MINVAL minimum value (default=0)
MAXVAL maximum value (default=1)
STEP step size (default=0.01)
--------------------------------------------------------------
### checkbox ###
SELECT 1/0 - checked(1), or unchecked (0) -> was idCHECKED before!!
--------------------------------------------------------------
### radio: ###
SELECT -1(none) 0...n
COUNT number of items
TEXT token1|token2|token3|...
CLICKED inform about SELECT change
--------------------------------------------------------------
### file ###
TEXT filename
FILTER filter for file-open dialog
CLICKED inform about file change
--------------------------------------------------------------
### list / combo ###
TEXT token1|token2|token3...
COUNT number of items
SELECT -1(none) 0...n
CLICKED inform about SELECT change
--------------------------------------------------------------
### text, singletext ###
SELSTART index of selection
SELEND index of end of selection
READONLY TRUE/FALSE - can the text be edited?
--------------------------------------------------------------
### tab ###
SELECT the selected tab (0=first)
CLICKED inform about SELECT change
--------------------------------------------------------------
### frame ###
TEXT caption text of the frame. Set to "" for an
invisible frame.
HEIGHT gets updated at every redraw.
WIDTH If =0 on creation, this will be set to the (one) widest child
on the first drawing. Beware of DDgui_Spacer(10000,0)!
